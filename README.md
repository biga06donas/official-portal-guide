# official-portal-guide

本仓库用于归档和发布多个独立 HTML 页面，内容涵盖各类信息展示与功能指引。所有页面均为静态文件，可直接通过浏览器打开，无需后端环境。

## 项目简介

这是一个通用型 HTML 页面归档仓库，主要用途包括：

- 存放与整理需要长期保留的网页快照或静态页面
- 为不同场景提供可复用的页面模板和参考材料
- 方便团队或个人对页面内容进行版本管理与协作更新

仓库不绑定任何特定域名或网站，所有页面内容均以独立文件形式存在，便于迁移和二次使用。

## 目录说明

仓库主要目录结构如下（可根据实际内容调整）：

```
/
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 页面引用的静态资源（CSS、JS、图片等）
├── archives/       # 历史版本或不再活跃的页面归档
└── README.md       # 本文件
```

- `pages/`：当前活跃的页面文件，按功能或主题命名子目录。
- `assets/`：各页面共享的样式、脚本及媒体资源。
- `archives/`：旧版本或已停止维护的页面，保留以备查阅。

## 页面归档说明

- 每个 HTML 页面应为一个完整、自包含的文档，尽量不依赖外部网络资源（除非必要）。
- 页面命名建议使用有意义的英文或拼音，避免特殊字符。
- 如需引用仓库内的静态资源，请使用相对路径。
- 归档时可在 `archives/` 目录下按日期或版本号建立子目录存放。

## 维护说明

- 欢迎提交 Issue 或 Pull Request 来更新、修正或新增页面。
- 修改前请确保页面在不同浏览器和屏幕尺寸下基本可用。
- 不建议在页面中嵌入跟踪代码、广告或外部动态内容。
- 仓库维护者会定期检查页面链接和资源引用是否有效。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 直接在浏览器中打开 `pages/` 目录下的任意 `.html` 文件即可查看。
3. 也可通过 GitHub Pages 等静态托管服务发布后在线访问（需自行配置）。
